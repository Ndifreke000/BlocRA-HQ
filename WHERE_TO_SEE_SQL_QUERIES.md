# Where to See SQL Queries in BlocRA

## Overview
SQL queries are visible in multiple places throughout the app, depending on what you're trying to do.

## 1. Query Editor (`/query`) - Main SQL Interface

### Query Input Area
**Location**: Center of the page, large textarea
**What you see**: 
- The SQL query you're currently writing or editing
- Syntax highlighting (basic)
- Line numbers (if enabled)
- Real-time validation feedback

**How to use**:
```
1. Go to http://localhost:8080/query
2. Type or paste your SQL query in the large text area
3. Click "Execute Query" to run it
4. Results appear below
```

**Example**:
```sql
SELECT block_number, timestamp, transaction_count 
FROM blocks 
ORDER BY block_number DESC 
LIMIT 10;
```

### AI-Generated SQL
**Location**: Same textarea, but generated by AI
**How to generate**:
1. Click "AI Assistant" tab
2. Enter a natural language prompt (e.g., "Show me the latest 10 blocks")
3. Click "Generate SQL"
4. AI-generated SQL appears in the query textarea
5. You can edit it before running

**Example Prompts**:
- "Show recent transactions with high fees"
- "Analyze block data for the last hour"
- "Find all contract interactions"

### Saved Queries Sidebar
**Location**: Right sidebar in Query Editor
**What you see**:
- List of your 5 most recent saved queries
- Query name
- Last run timestamp
- Quick actions (Load, Delete)

**How to access**:
1. Look at the right sidebar
2. See "Saved Queries" section
3. Click on any query name to load it into the editor
4. Click "View All Saved Queries" to see complete list

### Query History
**Location**: Accessible via keyboard shortcut
**What you see**:
- Your last 20 executed queries
- Ordered by most recent first

**How to access**:
- Press `Ctrl/Cmd + H` in Query Editor
- Or use up/down arrow keys to cycle through history

## 2. Dashboard Builder (`/builder`) - Using Saved Queries

### Saved Query Dropdown
**Location**: "Add Widget" dialog
**What you see**:
- Dropdown list of all your saved queries
- Query name
- Contract address (if applicable)
- Created date

**How to access**:
1. Go to http://localhost:8080/builder
2. Click "+ Add Widget"
3. See "Select Saved Query" dropdown
4. Click to see all your saved queries
5. Select one to use in your dashboard

**What's displayed**:
```
Query Name: "Daily Transactions"
Contract: 0x123...789
Created: 2024-12-30
```

**Note**: You DON'T see the actual SQL here - just the query metadata. The SQL is stored and executed when the widget runs.

### Widget Query Display
**Location**: Widget header (when hovering)
**What you see**:
- Widget title (which you named)
- Visualization type icon
- Play button to re-run query

**To see the actual SQL**:
- Currently NOT displayed in widget
- Stored in backend
- Executes when you click Play button

## 3. Library Page (`/library`) - Query Collection

### Query List View
**Location**: Main library page
**What you see**:
- All your saved queries
- Query name
- Description
- Created date
- Last run date
- Tags/categories

**How to access**:
1. Go to http://localhost:8080/library
2. Or click "View All Saved Queries" from Query Editor sidebar

**Actions available**:
- View query details
- Edit query
- Delete query
- Run query
- Add to dashboard

### Query Detail View
**Location**: Click on a query in library
**What you see**:
- Full SQL query text
- Query metadata (name, description, created date)
- Execution history
- Results from last run

## 4. Admin Dashboard (`/admin`) - Query Logs

### Query Logs Section
**Location**: Admin dashboard, Query Logs tab
**What you see**:
- All queries executed by all users
- User who ran the query
- Timestamp
- Query text (full SQL)
- Execution time
- Results count

**How to access**:
1. Go to http://localhost:8080/admin
2. Login with password: `Ndifreke000`
3. Click "Query Logs" tab
4. See all executed queries

**Admin only**: This is for monitoring and debugging purposes.

## 5. Browser Developer Tools - Network Tab

### API Requests
**Location**: Browser DevTools → Network tab
**What you see**:
- API calls to `/api/query/execute`
- Request payload with SQL query
- Response with results

**How to access**:
1. Open browser DevTools (F12)
2. Go to Network tab
3. Execute a query in Query Editor
4. Click on the `/api/query/execute` request
5. View "Payload" tab to see SQL
6. View "Response" tab to see results

**Example payload**:
```json
{
  "query": "SELECT * FROM blocks LIMIT 10",
  "contract_address": "0x123..."
}
```

## 6. Database Direct Access (Developers)

### SQLite Database
**Location**: `backend-rust/data/blocra.db`
**What you see**:
- All saved queries in `contract_queries` table
- Query text, name, user_id, created_at

**How to access**:
```bash
# From backend-rust directory
sqlite3 data/blocra.db

# View all saved queries
SELECT id, name, query, created_at FROM contract_queries;

# View specific user's queries
SELECT * FROM contract_queries WHERE user_id = 1;

# View query with results
SELECT q.name, q.query, q.created_at 
FROM contract_queries q 
ORDER BY q.created_at DESC;
```

## Common Use Cases

### "I want to see the SQL I just wrote"
→ Look in the Query Editor textarea (`/query`)

### "I want to see my saved queries"
→ Check the right sidebar in Query Editor or go to `/library`

### "I want to see what SQL a dashboard widget uses"
→ Currently not visible in UI - stored in backend
→ **Enhancement needed**: Add "View Query" button to widgets

### "I want to see all queries ever executed"
→ Admin Dashboard (`/admin`) → Query Logs tab

### "I want to see the SQL before adding to dashboard"
→ Go to Query Editor, load the saved query, view it there

### "I generated SQL with AI - where is it?"
→ It appears in the Query Editor textarea immediately after generation

## Tips

### To Review a Query Before Using in Dashboard:
1. Go to `/query`
2. Load the saved query from sidebar
3. Review the SQL
4. Test it by clicking "Execute Query"
5. If good, go to `/builder` and add it

### To See Query Results:
1. Execute query in Query Editor
2. Results appear in table below editor
3. Can export as CSV or JSON

### To Debug a Query:
1. Check Query Editor for syntax errors (red underline)
2. Check browser console for API errors
3. Check Admin Dashboard for execution logs
4. Check backend logs for detailed errors
